# Contributing

Thanks for taking the time to contribute.

## Guidelines
- Keep changes focused and scoped to a single topic.
- Use clear, concise English in documentation and commit messages.
- Update or add documentation when behavior or structure changes.
- Add or update tests when you introduce testable behavior.

## Development Notes
- There is no build system in this repository yet.
- If you add one, document the commands in `README.md`.

## Commit Messages
Use short, imperative messages such as "Add issue templates" or "Fix README".
